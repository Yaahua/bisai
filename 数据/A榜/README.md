## 关键经验总结

当前最重要的经验，不再是“多模型集成本身是否有效”，而是**哪一类增量可以建立在真实高分包之上继续加**。今日结果已经给出非常明确的反馈：

| 方向 | 结论 | 说明 |
| :--- | :--- | :--- |
| 在真基线上做减法清洗 | 失败 | `submit_v31_trim_none` 从 0.4449359 掉到 0.4445 |
| 在真基线上做小步纯加法 | 成功 | `submit_v32_plus_abs_var` 升到 0.4479 |
| 一次性补太多类型 | 高风险 | 关系均值上升太快，容易重走早期过标老路 |
| 回到旧的 `v17` 主线 | 不再合理 | 真实高分已证明后期有效底座并非 `v17` |

因此，后续提交应遵守一个更严格的原则：**绝不破坏 `submit_v30_fixed` 的原始结构，所有动作都应以“少量加法、优先高价值关系类型”为中心。**
