# 官方原始数据

本目录用于单独保存**来自比赛官方仓库的原始数据文件**，与仓库中用于实验、提交构造和结果产物的其他数据文件分开管理，避免混淆。

## 当前内容

| 文件 | 说明 |
| :--- | :--- |
| `train.json` | 官方原始训练集，未做任何 AI 修订、清洗或重写。 |
| `test_A.json` | 官方原始 A 榜测试集，未做任何改写，作为官方发布的测试输入保留。 |

## 设计原则

本目录强调“**原样保留**”。也就是说，这里的文件应始终被视为官方基准数据，后续如果需要做统计分析、Few-shot 检索、规则提取、误差诊断或提交复现，应当基于这里的文件读取，但不应直接在这里改写内容。

为了避免历史上出现过的“AI 按学术标准修正官方标注、导致训练标准偏移”的问题，本目录专门独立出来，目的就是把**官方原始数据**和**项目过程产物**彻底隔离。

## 与其他目录的关系

| 目录 | 作用 | 与本目录关系 |
| :--- | :--- | :--- |
| `数据/训练集/` | 项目运行时已接入的数据位置 | 可用于脚本兼容，但不强调“原始只读”语义 |
| `数据/A榜/` | 各版本提交 JSON / ZIP 与过程产物 | 属于项目输出目录，不等同于官方原始测试文件 |
| `分析报告/` | 基于官方数据做出的统计和诊断文档 | 其结论应可回溯到本目录的原始文件 |
