# 数据目录说明

本仓库当前仅对**数据部分**进行阶段区分，统一设置为“训练集”和“A榜”两个子目录。这样处理的目的，是将已经实际开展的训练数据标注成果与未来可能启动的 A 榜数据处理预留空间分开存放，同时保持经验总结、技能文件和通用说明等共享内容继续集中管理，避免重复维护。

| 子目录 | 当前状态 | 用途说明 |
|------|------|------|
| `训练集/` | 已启用 | 存放训练数据相关的切分方案、原始切分块、分类说明、审查文档、修订日志、修订结果、合并结果及验证报告。 |
| `A榜/` | 占位 | 仅保留目录结构，待用户后续正式启动 A 榜任务后再写入对应数据与过程文件。 |

后续若继续处理训练数据批次，应统一写入 `数据/训练集/` 下的既有结构；若未来启动 A 榜任务，则在 `数据/A榜/` 下按同类逻辑建立对应目录，而不应改动经验总结与技能等共享目录的归属。
