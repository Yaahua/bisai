# 已记录的训练集切分方式

依据对用户仓库既有修订结果目录的一次性检查，可确定当前训练数据切分方式如下：

| 项目 | 记录结果 |
|------|------|
| 文件目录 | `数据/训练集/修订结果/` |
| 文件命名 | `chunk_001_corrected.json` 至 `chunk_010_corrected.json` |
| 编号方式 | 三位数字顺序编号，左侧补零 |
| 每块条数 | 每个 `chunk` 含 10 条顶层样本 |
| 文件内容格式 | JSON 数组；每条样本含 `text`、`entities`、`relations` 三个核心字段 |
| 标注结构 | `entities` 使用 `start`/`end`/`text`/`label`；`relations` 使用 head/tail 文本、边界、类型和 `label` |
| 当前任务对应原则 | 继续沿用相同的“每块 10 条、三位编号、逐块 JSON 文件”方案，新增工作块应对应第 11–20 块 |

后续切分目标仓库数据时，按以上记录执行，不再回读用户该仓库。
