# CCL2026-MGBIE 杂粮育种信息抽取比赛方案

本仓库是针对天池 [CCL2026-MGBIE 杂粮育种信息抽取评测](https://tianchi.aliyun.com/competition/entrance/532465) 的完整解决方案，当前专注于 **Track-A（不微调赛道）**。整体路线已经从早期的静态 Few-shot 与单模型试探，迭代到 **RAG 动态检索、多模型集成、规则后处理、真实高分包反查，以及基于真实高分基线的小步加法微调**。

| **当前最高得分** | **0.4479** |

## 核心技术演进

### 1. 单模型试探到 RAG 动态检索

早期版本通过静态 Few-shot 与提示词约束进行试探，随后引入 TF-IDF 检索为每条测试文本动态匹配训练样本，显著改善了示例覆盖与关系抽取稳定性。以 `v7_cicl_v2` 为代表的这一路线，将单模型成绩提升到 **0.3746**，并为后续集成提供了高质量底座。

### 2. 多模型集成成为第一次跃迁

在 `v7`、`v9`、`v10` 三个风格差异明显的模型结果上执行多数投票后，`ensemble_v3` 将分数提升到 **0.4172**。这一步的本质并不是“多就是好”，而是利用异质模型的交集来压制独立假阳性，从而获得极高精度。

### 3. 极保守白名单后处理带来 0.4224

在 `ensemble_v3` 基础上，`v17_whitelist` 用非常克制的规则补充了少量高置信关系，使仓库内公开路线的最高分来到 **0.4224**。当时这一结果一度被视作主线，因此围绕 `v18-v21` 产生了大量加法、验证器和再集成实验。

### 4. 对真实高分线的误判被纠正

后续实战证明，仓库中公开记录并没有覆盖用户手里真正的高分提交。通过反查用户上传的 `submit_v30_fixed.zip`，可以确认它与 `submit_v29_postprocessed.json` 最为接近，而且比 `v29` **额外多出 110 条关系**，并且这些增量并不等价于此前自动生成的 `v30_safe` 或 `v30_ultrasafe`。这一步纠正了整个仓库的主分析方向。

### 5. 真基线上的小步加法产生新高分

在确认 `submit_v30_fixed` 才是真正高分底座后，仓库放弃了“先做减法清洗再看结果”的主思路，转而围绕真基线做 **小步、纯加法** 微调。今日的 `submit_v31_trim_none` 提交得到 **0.4445**，说明减法会破坏原始高分结构；随后 `submit_v32_plus_abs_var` 只额外补入 `ABS-AFF-TRT` 与 `VAR-HAS-TRT` 两类关系，共 **56 条**，成绩提升到 **0.4479**，成为当前仓库主线。

## 当前主线结论

| 路线 | 结论 | 当前判断 |
| :--- | :--- | :--- |
| `v17_whitelist` 底座继续做大加法 | 已被后续真实成绩否定 | 不再作为主线 |
| `submit_v30_fixed` 真基线做减法清洗 | 今日实测掉到 0.4445 | 仅作诊断，不宜继续 |
| `submit_v30_fixed` 真基线做小步加法 | 今日实测提升到 0.4479 | **当前唯一主线** |
| 大幅增加 `QTL/GENE` 类补关系 | 风险较高 | 仅作后备试验 |

## 快速开始

### 1. 运行 v21 精英版单模型预测

如果需要在当前官方数据路径下重跑单模型预测，可直接运行：

```bash
python3.11 脚本工具/predict_v21_elite.py
```

### 2. 复现 v29 后处理底座

如果需要回到真实高分线最接近的公开底座，可运行：

```bash
python3.11 脚本工具/ensemble_v29.py
python3.11 脚本工具/postprocess_v29.py
```

### 3. 构造真基线上的 v31 / v32 候选

围绕 `submit_v30_fixed.json` 继续生成小步候选时，可运行：

```bash
python3.11 脚本工具/make_v31_from_true_best.py
python3.11 脚本工具/make_v32_small_additions.py
```

### 4. 一键打包提交

```bash
python3.11 脚本工具/make_submit_zip.py 数据/A榜/submit_v32_plus_abs_var.json
```
