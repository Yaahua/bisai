# CCL2026-MGBIE 杂粮育种信息抽取比赛方案

本仓库是针对天池 [CCL2026-MGBIE 杂粮育种信息抽取评测](https://tianchi.aliyun.com/competition/entrance/532465) 的完整解决方案。当前专注于 **Track-A（不微调赛道）**，通过深度分析官方数据规律、构建比赛级 Prompt 和精准后处理规则，实现高分预测。

## 🏆 当前成绩
- **最佳得分**: 0.3482 (A榜第3名)
- **最新版本**: v4 (预期突破 0.350)

## 📂 仓库结构

本仓库经过全面重构，结构清晰，分为以下几个核心模块：

| 目录 | 说明 | 核心内容 |

| [`数据/`](数据/README.md) | 比赛数据与提交文件 | 包含 A 榜历次提交的 `submit.zip` 及预测结果 JSON |

| [`分析报告/`](分析报告/README.md) | 深度数据分析与策略 | 包含赛道分析、得分诊断、冲分规划及可视化图表 |

| [`脚本工具/`](脚本工具/README.md) | 数据处理与可视化 | 包含用于提取规律、计算得分、生成图表的各类 Python 脚本 |

## 🚀 快速开始

### 1. 提交最新结果
直接下载 [`数据/A榜/submit_v4.zip`](数据/A榜/submit_v4.zip) 并提交至天池 Track-A 赛道。

### 2. 运行预测脚本
如果你想自己重新跑一遍预测：
```bash
# 运行 v2 版多线程预测脚本（带内置后处理）
python3 提示词库/predict_track_a_v2.py
```

### 3. 运行后处理过滤
如果你有新的预测结果，想过滤掉非法关系：
```bash
python3 提示词库/post_process_v4.py
```

## 📈 演进历程

- **v1 (基础版)**: 基于官方 5 条 Few-shot 样本的直接预测。
- **v2 (激进版)**: 增加严格 Prompt 约束，但导致严重漏标，分数下降。
- **v3 (稳健版)**: 在 v1 基础上仅删除 95 个训练集中不存在的非法三元组，得分 **0.3482**。
- **v4 (精准版)**: 在 v3 基础上进一步删除 179 个高概率错误关系（距离限制+关键词验证），预期突破 0.350。

## 💡 核心策略

1. **放弃修改训练集**：官方数据即 Ground Truth，不要用学术标准去纠正官方的标注偏好（如 LOI 和 AFF 的界定）。
2. **精准 Few-shot**：从官方训练集中提取覆盖所有关系类型的典型样本，特别是易漏标的 `GENE-LOI-TRT`。
3. **后处理兜底**：利用训练集的统计规律，强制删除预测结果中不可能存在的非法三元组（如 `BM-USE-CROP`）。
