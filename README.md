## 核心技术演进

### 1. 单模型试探到 RAG 动态检索

早期版本通过静态 Few-shot 与提示词约束进行试探，随后引入 TF-IDF 检索为每条测试文本动态匹配训练样本，显著改善了示例覆盖与关系抽取稳定性。以 `v7_cicl_v2` 为代表的这一路线，将单模型成绩提升到 **0.3746**，并为后续集成提供了高质量底座。

### 2. 多模型集成成为第一次跃迁

在 `v7`、`v9`、`v10` 三个风格差异明显的模型结果上执行多数投票后，`ensemble_v3` 将分数提升到 **0.4172**。这一步的本质并不是"多就是好"，而是利用异质模型的交集来压制独立假阳性，从而获得极高精度。

### 3. 极保守白名单后处理带来 0.4224

在 `ensemble_v3` 基础上，`v17_whitelist` 用非常克制的规则补充了少量高置信关系，使仓库内公开路线的最高分来到 **0.4224**。

### 4. 对真实高分线的误判被纠正

通过反查用户上传的 `submit_v30_fixed.zip`，确认它与 `submit_v29_postprocessed.json` 最为接近，且比 `v29` 额外多出 110 条关系。这一步纠正了整个仓库的主分析方向。

### 5. 真基线上的小步加法产生新高分（当前主线）

围绕 `submit_v30_fixed` 做**小步、纯加法**微调：

- `submit_v32_plus_abs_var`：+ABS-AFF-TRT:29 + VAR-HAS-TRT:27 → **0.4479**
- `submit_v36_gene_abs`：在 v32 基础上 +ABS-AFF-GENE:33 → **0.4487**（当前最高）

### 6. LLM 辅助补充（新探索，进行中）

使用 `gpt-4.1-mini` 对 test_A 中 111 个无关系样本进行关系抽取（POSW-Vote 3轮聚合），过滤后仅 6 条 3/3 一致关系，提交后得 0.4481（掉分）。说明无关系样本本身难度极高，LLM 补充效果有限。

**下一步**：转向对**已有关系但覆盖不全**的样本进行 LLM 补充，以及减法策略（删除假阳性）。
