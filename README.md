# CCL2026-MGBIE 杂粮育种信息抽取比赛方案

本仓库是针对天池 [CCL2026-MGBIE 杂粮育种信息抽取评测](https://tianchi.aliyun.com/competition/entrance/532465) 的完整解决方案。当前专注于 **Track-A（不微调赛道）**，通过 RAG 动态检索、多模型集成（Ensemble）以及纯规则后处理，实现了高分预测。

## 🏆 当前成绩
- **最佳得分**: **0.4172** (A榜并列第一名)
- **最新版本**: `v16_rules` (基于 ensemble_v3 的纯规则召回补充)

## 🚀 核心技术演进 (突破 0.4 路径)

### 1. 单模型瓶颈与 RAG 动态提示词 (v6-v7)
- **问题**：固定 Few-shot 样本导致模型产生大量幻觉，无法应对多样化的测试文本。
- **方案**：引入 TF-IDF 检索，为每条测试文本动态匹配 Top-4 最相似的训练样本（RAG）。同时引入代码风格提示词（c-ICL）。
- **结果**：单模型（GPT-4.1-mini）得分达到 **0.3746**。

### 2. 异质多模型多数投票集成 (ensemble_v3)
- **问题**：单模型（即使是 GPT-4）存在固有的独立错误和幻觉，无法通过 Prompt 完全消除。
- **方案**：使用 GPT-4.1-mini（v7, v9）和 Gemini-2.5-flash（v10）生成三份独立的预测结果，进行多数投票（MIN_VOTES=2）取交集。
- **结果**：系统性消除了假阳性（FP），得分飙升至 **0.4172（并列第一）**。

### 3. 突破集成瓶颈：纯规则精准召回 (v16)
- **问题**：多模型集成虽然极大提升了精确率，但导致召回率断崖式下降（关系均值仅 2.16，远低于官方期望的 2.80）。
- **方案**：通过分析发现大量"无关系"句子实际上提取了实体，但漏标了关系。编写纯规则脚本（零 API 消耗），当实体间的文本包含明确触发词（如 `including`, `associated with`）时，强制补充关系。
- **结果**：精准补充了 238 条高置信度关系，均值回升至 2.75，完美拟合官方分布。

## 🛠️ 快速开始

### 1. 运行单模型预测
```bash
# 运行 GPT-4.1-mini (RAG 动态检索版)
python3.11 提示词库/predict_track_a_v7_cicl_v2.py

# 运行 Gemini-2.5-flash
python3.11 提示词库/predict_track_a_v10_gemini.py
```

### 2. 运行多模型集成
准备好多个模型的预测结果后，运行多数投票脚本：
```bash
python3.11 提示词库/ensemble_vote.py
```

### 3. 运行纯规则后处理
对集成结果进行召回率补充：
```bash
python3.11 提示词库/postprocess_v3_rules.py
```

### 4. 一键打包提交
```bash
python3.11 脚本工具/make_submit_zip.py 数据/A榜/submit_v16_rules.json
```
