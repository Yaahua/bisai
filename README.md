## 🚀 核心技术演进与诊断

### 1. 单模型瓶颈与 RAG 动态提示词 (v6-v7)
早期尝试中，固定 Few-shot 样本导致模型产生大量幻觉。引入 TF-IDF 检索为每条测试文本动态匹配 Top-4 最相似的训练样本（RAG）后，单模型（GPT-4.1-mini）得分达到 **0.3746**。

### 2. 异质多模型多数投票集成 (ensemble_v3)
为消除单模型的独立错误和幻觉，使用 GPT-4.1-mini（v7, v9）和 Gemini-2.5-flash（v10）生成三份独立的预测结果，进行多数投票（MIN_VOTES=2）取交集。该策略系统性消除了假阳性（FP），将得分推高至 **0.4172**。

### 3. 极保守白名单后处理 (v17_whitelist)
多模型集成虽然极大提升了精确率，但导致召回率断崖式下降。通过对训练集触发词进行深度统计分析，我们在 v3 基础上补充了 `QTL on chromosome` 等预期精确率 $\ge$ 90% 的保守规则，在不损失精确率的前提下将得分提升至 **0.4224（当前最高分）**。

### 4. "加法策略"的破产与全量诊断 (v18-v20)
在尝试进一步突破 0.43 的过程中，我们执行了白名单扩展（v18）、自一致性采样（SC）以及多源交叉验证合并（v20 系列）。然而，**实际提交结果表明，在弱模型集成的基础上做任何形式的"加法"都无法显著提分**。新增关系的精确率极低（约 1%），大量假阳性涌入导致 F1 增益几乎为零甚至为负。这证明了"在弱模型上做加法"的策略已经走到死胡同。

## 🛠️ 快速开始

### 1. 运行精英版单模型预测（最新策略）
完全放弃多模型集成，用极强 Prompt + RAG + 自一致性构建高质量单模型预测：
```bash
python3.11 脚本工具/predict_v21_elite.py
```

### 2. 运行经典多模型集成
准备好多个模型的预测结果后，运行多数投票脚本：
```bash
python3.11 提示词库/ensemble_vote.py
```

### 3. 运行纯规则后处理
对集成结果进行保守规则补充：
```bash
python3.11 脚本工具/postprocess_v17_whitelist.py
```

### 4. 一键打包提交
```bash
python3.11 脚本工具/make_submit_zip.py 数据/A榜/submit_v17_whitelist.json
```
