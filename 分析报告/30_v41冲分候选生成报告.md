## 路径一：减法策略（分析底座假阳性并删除）

### 方法

通过多维信号综合评分，识别底座中可能的假阳性关系：

| 信号 | 权重 | 说明 |
| :--- | :--- | :--- |
| between 词不匹配 | +2 | 两实体之间的文本特征与训练集模式不符 |
| 实体距离异常 | +1~2 | 距离超过训练集同类型的 1.5~2 倍标准差 |
| 泛称实体 | +2 | head/tail 是泛称词（如 varieties, genotypes, QTLs） |
| 实体文本过短 | +1 | 实体文本 ≤2 字符（非 CHR 类型） |

## 策略分析

### 为什么减法可能有效

当前底座 rel_avg=2.70，接近训练集期望值 2.80，但无关系比例 27.8% 低于训练集的 32.7%。这说明底座可能存在过度抽取的问题——部分本应无关系的样本被错误地添加了关系。保守减法（删除高置信假阳性）可以提升精确率而不过度损失召回率。

### 为什么交叉验证加法最安全

交叉验证加法只保留在旧模型（v30_safe/v30_supported_alltypes）中也出现的候选。这些候选经过了多个独立模型的"背书"，假阳性概率最低。虽然数量少（仅10条），但每条的可靠性极高。

### 风险评估

| 策略类型 | 涨分概率 | 掉分风险 | 涨分幅度 |
| :--- | :--- | :--- | :--- |
| 保守减法 | 60% | 低 | +0.001~0.005 |
| 交叉验证加法 | 50% | 极低 | +0.001~0.003 |
| 减法+加法组合 | 55% | 低 | +0.002~0.008 |
| 大幅减法（泛称） | 40% | 中 | +0.005~0.015 或 -0.005 |
| 大量规则加法 | 30% | 高 | +0.005 或 -0.010 |
