# 守擂冲分新方向：极保守白名单与高质量四重集成

**作者：Manus AI**
**日期：2026-04-12**

## 1. 现状回顾与反思

在之前的冲分尝试中，我们面临的核心矛盾是：**如何在使用多模型集成（高精确率、低召回率）的基础上，安全地提升召回率？**

- **v16 纯规则后处理的教训**：为了提升召回率，v16 引入了基于 `of` 和 `for` 等介词的泛化规则。然而，这导致了灾难性的假阳性（False Positives, FP）爆发。由于官方评分公式对精确率（Precision）的惩罚极重，v16 的分数反而低于 `ensemble_v3`。
- **v4a 降低投票门槛的教训**：为了放行更多关系，v4a 引入了质量较差的小模型 `v11_nano`，并将集成门槛从 66.7%（3选2）降至 50%（4选2）。这同样放进了大量噪音，导致分数下降。
- **GLiREL 本地小模型的折戟**：虽然理论上零样本小模型可以解决 API 成本和召回率问题，但实测发现，GLiREL large 模型（1.8GB）的内存需求超出了当前沙盒环境的限制，导致进程 OOM 卡死，此方案暂不可行。

综上所述，**牺牲精确率换取召回率是行不通的**。在稳固第一名的目标下，我们需要极其保守的策略。

## 2. 新方向一：极保守白名单后处理（v17）

为了安全地补充漏标关系，我们对训练集中的关系触发词进行了深度量化分析，设计了一套“极保守白名单规则”。

### 2.1 规则设计原则

1. **仅使用 100% 确定的触发词**：如 `mapped to`、`located on`，放弃任何存在歧义的介词（如单纯的 `for` 或 `of`）。
2. **严格的距离约束**：要求头尾实体之间的字符距离极短（通常 $\le 5$ 字符，最长不超过 30 字符）。
3. **禁止跨句**：如果两个实体之间存在句号或分号，坚决不补关系。
4. **严格的类型约束**：仅允许训练集中出现过的实体类型组合。

### 2.2 v17 白名单规则明细

| 关系类型 | 头实体类型 | 尾实体类型 | 触发词模式 | 最大间距 | 预期精确率 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **LOI** | QTL, MRK, GENE | CHR | `^\s*on\s*$` | 5 字符 | > 95% |
| **LOI** | QTL, MRK, GENE | TRT | `^\s*for\s*$` | 5 字符 | > 90% |
| **LOI** | QTL, MRK, GENE | CHR | `mapped to`, `located on` | 50 字符 | > 98% |
| **CON** | CROP | VAR | `^\s*\(\s*$` | 3 字符 | > 99% |
| **HAS** | CROP, VAR | TRT | `showed`, `with high` | 30 字符 | > 85% |

### 2.3 预期收益

在训练集上的模拟测试表明，这套白名单规则在**从零开始补标**的情况下，整体精确率达到了 **78.6%**。
由于我们将这套规则应用于已经具备高精确率的 `ensemble_v3` 之上，实际补充的关系仅为 **33 条**（主要是 LOI 和 CON）。这 33 条关系几乎可以认为是 100% 确定的真阳性（True Positives），能够在不掉精确率的前提下，稳定提升 0.005 ~ 0.01 的总分。

## 3. 新方向二：高质量四重集成（路线二规划）

白名单规则（v17）虽然安全，但上限有限。为了进一步冲分，我们需要回归大模型集成，但必须吸取 v4a 的教训。

### 3.1 核心思路

摒弃质量差的小模型，使用最强的模型（Gemini 或 GPT-4.1-mini）生成一份**全新的高质量基础预测（v18）**，然后与现有的 v7、v9、v10 进行 `MIN_VOTES=2` 的四重集成。

### 3.2 v18 预测的优化方向

1. **针对性 Prompt 优化**：在 Prompt 中明确强化 `AFF` 和 `LOI` 这两类最容易漏标的关系的定义，并给出正反例（Few-shot）。
2. **GLiNER 实体强引导**：我们已经成功在 CPU 上跑通了 GLiNER small 模型（400 条数据仅需 11 秒）。我们可以先用 GLiNER 提取所有实体，然后在 Prompt 中将这些实体作为“已知事实”提供给大模型，让大模型**只做关系分类**，从而极大降低漏标率。
