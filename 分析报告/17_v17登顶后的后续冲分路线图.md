# v17 登顶后的后续冲分路线图

**作者：Manus AI**
**日期：2026-04-12**

## 1. 核心战果与策略复盘

在 2026-04-12 凌晨的冲分中，我们通过 **v17（极保守白名单后处理）** 方案，在 `ensemble_v3`（0.4172 分）的基础上，仅补充了 **33 条高置信度关系**，成功将分数提升至 **0.4224**，登顶 A 榜第一。

### 1.1 经验总结
1. **精确率是命脉**：官方评分公式对假阳性（FP）的惩罚极其严厉。v16（纯规则补充 238 条）和 v4a（降门槛补充 185 条）的失败，证明了“牺牲精确率换取召回率”是稳亏不赚的买卖。
2. **“少而精”胜过“大而全”**：v17 成功的核心在于**极度克制**。我们通过对训练集的量化分析，只保留了精确率预期 >85% 的触发词（如 `mapped to`、`CROP (VAR)`），并施加了严苛的距离限制（$\le 5$ 字符）和防跨句限制。

## 2. 后续冲分路线规划

虽然目前已取得第一，但为了拉开差距并应对其他选手的追赶，我们制定了以下三条递进式的冲分路线，供后续参考与执行。

### 路线 A：继续深挖“白名单”红利（零 API 成本，最稳妥）
v17 的成功证明了这条路是通的。目前 v17 只覆盖了最容易提取的 LOI、CON 和 HAS 关系，对 AFF 关系的处理非常保守。

**具体行动：**
1. **深挖 AFF 触发词**：分析训练集中 `regulates`、`controls` 等动词的上下文，结合句法结构（如主动语态 vs 被动语态），将 AFF 的白名单精确率提升至 90% 以上。
2. **挖掘 USE 关系**：训练集中 `USE` 关系（使用某种育种方法）往往伴随 `using`、`by means of` 等明确介词，目前 v17 完全没有覆盖。
3. **挖掘 OCI 关系**：分析 `at the [stage] stage` 的具体模式，补充缺失的生长阶段关系。

### 路线 B：GLiNER 实体引导的大模型推理（需 API 额度，上限更高）
大模型（如 Gemini / GPT-4.1-mini）之所以漏标关系，往往是因为它在第一步就**漏标了实体**。既然我们已经验证了 GLiNER small 在 CPU 上运行极快（400 条仅需 11 秒），我们可以采用**两阶段流水线**。

**具体行动：**
1. **阶段一（本地零成本）**：使用 GLiNER small 提取测试集中的所有实体（NER）。
2. **阶段二（大模型 API）**：构建全新的 Prompt，将 GLiNER 提取的实体作为“已知事实”直接喂给大模型。
   - *Prompt 示例*：“我已经帮你找到了这些实体：[实体列表]。请你只负责判断这些实体之间是否存在以下关系，不要自己去寻找新实体。”
3. **优势**：极大降低了大模型的任务复杂度，有望在保持高精确率的同时，大幅提升关系召回率。

### 路线 C：高质量的“四重集成”（终极杀器）
如果路线 B（GLiNER 引导方案）能产出一份高质量的预测结果（我们称之为 v18），那么我们就可以将其与现有的高分预测结合，打造终极集成方案。

**具体行动：**
1. 收集目前的优质预测文件：`v7`、`v9`、`v10`，以及新产出的 `v18`。
2. 执行 `MIN_VOTES=2` 的四模型集成。
3. **防噪音机制**：为了避免重蹈 v4a 的覆辙，必须在集成前单独评估 v18 的精确率。只有当 v18 的单模型精确率接近 v9 时，才能将其纳入集成池。

---

## 3. 优先级建议

- **当前阶段**：以**路线 A** 为主。不需要消耗任何 API 额度，纯靠代码分析训练集，不断打磨白名单规则脚本，每次只补最确定的几十条关系，稳步推高分数。
- **冲刺阶段**：当 API 额度充足时，启动**路线 B**，生成 v18。
- **决战阶段**：在比赛截止前，执行**路线 C**，提交终极四重集成版本。
