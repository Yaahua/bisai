# 守擂与长远冲分规划：突破模型集成瓶颈的精细化策略

## 1. 当前瓶颈分析：为什么“堆模型”不可持续？

在之前的冲分过程中，我们通过“多模型多数投票（Majority Voting）”成功登顶（0.4172）。但随着模型数量的增加，我们观察到了明显的**边际收益递减**和**召回率断崖式下降**。

通过对 `ensemble_v3` 和 `ensemble_v4a/v4b` 的深度统计分析，我们发现：
- **取交集（AND）或高阈值投票（MIN_VOTES=3）**：精确率极高，但召回率损失惨重。例如 `v4b` 的无关系比例高达 54.8%（期望值为 32.7%），大量合法的低频关系（如 `VAR-HAS-TRT`）被丢弃。
- **取并集（OR）或低阈值投票（MIN_VOTES=1/2）**：召回率恢复，但 API 限速和个别模型的“幻觉”会引入噪声，导致精确率下降。
- **API 稳定性问题**：新增模型（如 Gemini、GPT-4.1-nano）在并发预测时容易触发 429 限速，导致数据包缺失，修复成本高昂。

单纯的“堆模型”已经触及天花板。为了稳固第一名并为 B 榜做准备，我们需要从“粗放式集成”转向**“精细化推理与置信度校准”**。

## 2. 学术前沿支撑：2025-2026 年 IE 领域最新突破

基于最新的学术研究，我们梳理了三条可行的精细化突破路径：

### 2.1 动态置信度校准与自适应加权集成 (Confidence-Aware Adaptive Ensemble)
传统的多数投票假设所有模型在所有样本上的可靠性相同。然而，最新研究表明，通过校准 LLM 的输出置信度，可以显著提升集成的 F1 分数 [1] [2]。
- **核心思想**：不再使用一刀切的 `MIN_VOTES`，而是根据模型在特定关系类型上的历史表现（全局可信度）和当前预测的概率/一致性（局部置信度）进行加权投票。
- **实施方案**：利用 Self-Consistency（自一致性）采样 [3]，让单一模型对同一句子预测 3 次，计算实体和关系的内部一致性分数。高置信度的预测直接保留，低置信度的预测交由更强的模型（如 GPT-4）进行二次确认。

### 2.2 自纠正迭代精化 (Self-Correcting Iterative Refinement, SCIR)
单次 Prompt 预测容易遗漏复杂关系。2025 年提出的 SCIR 框架 [4] 证明，通过多轮迭代和自我反思，可以大幅降低漏标率。
- **核心思想**：将抽取过程分为“初次抽取 → 缺失检测 → 冗余检测 → 最终输出”的闭环。
- **实施方案**：在预测脚本中引入“反思 Prompt”。当模型输出结果后，不直接采纳，而是将结果和原文再次输入给模型，询问：“请检查是否遗漏了 CROP 实体？是否存在非法的 TRT-HAS-VAR 关系？”。这种基于 Schema 的自我纠正能有效提升精确率。

### 2.3 多智能体协同与实体消歧 (Multi-Agent Disambiguation)
在低资源和垂直领域（如农业育种），LLM 经常混淆实体边界或类型（例如将基因名误认为品种名）。KDR-Agent 框架 [5] 提出了一种有效的多智能体协同机制。
- **核心思想**：将 NER 和 RE 拆分，并引入专门的“消歧智能体（Disambiguation Agent）”和“知识检索智能体（Knowledge Retrieval Agent）”。
- **实施方案**：遇到罕见实体（如特定的 QTL 或 Marker），先调用检索工具或外部知识库获取其定义，生成上下文解释，再指导抽取智能体进行分类。

## 3. 三阶段守擂与跃进执行计划

为了在保证当前优势的前提下稳步提升，我们制定了以下三阶段执行计划：

### 阶段一：实现置信度感知的自适应集成 (1-2周)
**目标**：解决 `ensemble_v4` 中召回率和精确率的硬权衡问题。
**技术手段**：
1. **废弃静态 `MIN_VOTES`**：编写 `adaptive_ensemble.py`。
2. **关系级别加权**：根据训练集统计，赋予不同模型在不同关系上的权重。例如，v7 在 `QTL-LOI-TRT` 上表现最准，则赋予更高权重；Gemini 在实体召回上表现好，则在 NER 阶段赋予更高权重。
3. **引入 Self-Consistency 分数**：对分歧较大的句子，使用强模型（GPT-4o）进行 Tie-breaker（决胜投票）。

### 阶段二：引入 SCIR 自纠正机制 (2-3周)
**目标**：降低单模型的幻觉和漏标，提升单模型输出质量，从而减轻集成阶段的压力。
**技术手段**：
1. **两阶段 Prompting**：
   - 第一轮：常规 RAG + c-ICL 抽取。
   - 第二轮：将第一轮结果作为输入，附带 `ILLEGAL_TRIPLETS` 黑名单和“易漏标实体（如 CROP）”提示，强制模型进行自我检查和修正。
2. **降低并发，保障稳定**：为了应对 API 限速，引入指数退避重试机制和更细粒度的断点续传。

### 阶段三：构建农业领域多智能体协同框架 (长远规划，备战 B 榜)
**目标**：彻底解决复杂长句和生僻实体的抽取难题。
**技术手段**：
1. **NER 智能体**：负责初步实体识别。
2. **消歧智能体**：检测到易混淆实体（如缩写、多义词）时，结合上下文生成解释。
3. **RE 智能体**：基于消歧后的实体列表，进行关系判断。
4. **反思智能体**：全局审查三元组的合法性和逻辑一致性。

## 4. 预期收益与风险控制

| 策略 | 预期 F1 提升 | 风险点 | 应对措施 |
| :--- | :--- | :--- | :--- |
| 自适应加权集成 | +0.01~0.015 | 权重过拟合 A 榜测试集 | 使用训练集进行交叉验证确定权重 |
| SCIR 自纠正 | +0.015~0.02 | API 成本翻倍，时间延长 | 仅对置信度低（模型间有分歧）的句子触发自纠正 |
| 多智能体协同 | +0.02+ | 流程复杂，容易出错 | 模块化开发，确保每个智能体可独立测试 |

通过实施上述精细化策略，我们不仅能稳固当前的并列第一，还能在 B 榜（通常数据分布更复杂、更具挑战性）中占据绝对的技术优势。
