## 第一步：运行 v6（RAG 动态提示词）

**原理**：放弃固定的 6 个 Few-shot 样本，改为对每条测试文本实时检索最相似的 5 条训练样本。

```bash
cd /home/ubuntu
python3.11 bisai/提示词库/predict_track_a_v6_rag.py
```

**运行时间**：约 5~10 分钟（400 条 × 20 线程）

**输出**：`/home/ubuntu/bisai/数据/A榜/submit_v6_rag.json`

**打包提交**：
```bash
python3.11 bisai/脚本工具/make_submit_zip.py bisai/数据/A榜/submit_v6_rag.json
```

**提交文件**：`bisai/数据/A榜/submit_v6_rag.zip`

**输出**：`/home/ubuntu/bisai/数据/A榜/submit_v7_cicl.json`

## 第三步（可选）：运行 v8（两阶段管道）

**原理**：将 NER 和 RE 拆分为两次独立的 API 调用。RE 阶段只在已识别的实体对之间判断关系，大幅降低任务复杂度。

```bash
python3.11 bisai/提示词库/predict_track_a_v8_pipeline.py
```

**注意**：每条文本需要 2 次 API 调用，总调用量约 800 次，费用约为 v6/v7 的 2 倍。

**输出**：`/home/ubuntu/bisai/数据/A榜/submit_v8_pipeline.json`

## 第四步（终极冲分）：多模型集成投票

**前提**：已完成 v6 和 v7（或 v8）的预测，生成了至少 2 个预测文件。

**原理**：对每个实体/关系三元组，只有当 2 个以上模型都预测出该结果时，才纳入最终提交。这能极其有效地过滤掉单个模型的随机幻觉。

**配置**（编辑 `ensemble_vote.py` 第 44~49 行）：
```python
PREDICTION_FILES = [
    '/home/ubuntu/bisai/数据/A榜/submit_v6_rag.json',
    '/home/ubuntu/bisai/数据/A榜/submit_v7_cicl.json',
    # 如有第三个模型，在此添加
]
MIN_VOTES = 2  # 2个文件时用1（OR合并），3个文件时用2（多数投票）
```

**运行**：
```bash
python3.11 bisai/提示词库/ensemble_vote.py
```

**输出**：`/home/ubuntu/bisai/数据/A榜/submit_ensemble.json`

**打包提交**：
```bash
python3.11 bisai/脚本工具/make_submit_zip.py bisai/数据/A榜/submit_ensemble.json
```

## 进阶技巧：引入第三个模型（强烈推荐）

为了让多数投票发挥最大效果，建议引入一个与 `gpt-4.1-mini` 架构不同的模型。

**方案 A：使用 Gemini（已在 OpenAI 兼容接口中可用）**

在 `predict_track_a_v6_rag.py` 或 `predict_track_a_v7_cicl.py` 中，将 `MODEL` 改为：
```python
MODEL = "gemini-2.5-flash"
```
重新运行，输出保存为 `submit_v6_gemini.json`，加入集成投票。

**方案 B：使用 GPT-4.1（更强的模型）**

```python
MODEL = "gpt-4.1"
```
费用更高，但单模型效果更好，可以直接作为 v9 提交。

## 评分公式参考

官方评分公式（用于估算提交分数）：

```
Score_NER = 0.5 × F1_NER + 0.25 × P_NER + 0.25 × R_NER
Score_RE  = 0.5 × F1_RE  + 0.25 × P_RE  + 0.25 × R_RE
Total     = 0.4 × Score_NER + 0.6 × Score_RE
```

**关键洞察**：RE 占总分的 60%，NER 占 40%。因此优化关系抽取的精确率和召回率，比优化实体识别的收益更大。

## 常见问题

**Q：v6 比 v5 慢很多？**
A：v6 每条文本都需要进行 TF-IDF 向量化检索，但检索本身极快（毫秒级），不会显著增加运行时间。主要瓶颈仍是 API 调用延迟。

**Q：如果 v6 分数反而下降怎么办？**
A：说明当前测试集与训练集的语义分布差异较大，TF-IDF 检索到的样本相关性不足。可以尝试：
1. 将 `TOP_K` 从 5 增加到 8
2. 改用 `sentence-transformers` 的语义向量替代 TF-IDF（更准确但更慢）

**Q：集成投票后分数反而下降？**
A：可能是 `MIN_VOTES=2` 过于严格，删除了太多正确的关系（Recall 下降）。尝试将 `MIN_VOTES` 改为 1（OR 合并，取两个模型预测的并集）。
